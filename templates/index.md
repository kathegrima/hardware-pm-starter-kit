# Templates Directory - README

> Quick reference guide for hardware project management templates

---

## Available Templates

| Template | Purpose | When to Use | File |
|----------|---------|-------------|------|
| **Project Charter** | Define project scope, objectives, constraints | Project kickoff | [01-project-charter.md](./01-project-charter.md) |
| **Risk Register** | Track and mitigate hardware-specific risks | Ongoing throughout project | [02-risk-register.md](./02-risk-register.md) |
| **Phase Gate Checklist** | Gate review criteria for Concept → EVT → DVT → PVT → Production | Before each phase transition | [03-phase-gate-checklist.md](./03-phase-gate-checklist.md) |
| **Sprint Planning** | Plan hardware sprints with lead-time tracking | Every 4-8 week sprint | [04-sprint-planning.md](./04-sprint-planning.md) |
| **Lessons Learned** | Capture project insights for continuous improvement | Project completion or major milestones | [05-lessons-learned.md](./05-lessons-learned.md) |
| **PRD (Product Requirements)** | Define product scope, specs, and requirements before development | Project kickoff (Concept phase) | [06-prd-template.md](./06-prd-template.md) |
| **BOM (Bill of Materials)** | Track components, costs, suppliers, and inventory | Throughout project (EVT → MP) | [07-bom-template.md](./07-bom-template.md) |
| **Compliance Checklist** | Track EU compliance requirements (CE, RoHS, RED, WEEE) | Pre-project planning through launch | [08-compliance-checklist.md](./08-compliance-checklist.md) |
| **Test Plan** | Define testing strategy, procedures, and pass/fail criteria | EVT phase (testing starts) | [09-test-plan-template.md](./09-test-plan-template.md) |
| **Change Request (ECO)** | Formal change control for hardware design changes | Post-design freeze (DVT exit) | [10-change-request-form.md](./10-change-request-form.md) |
